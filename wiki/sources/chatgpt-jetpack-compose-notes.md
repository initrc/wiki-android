## Key Claims

- Compose is declarative: update state, then Compose recomposes UI that reads that state.
- `Column`, `Row`, and `Box` are the basic primitives for vertical stacking, horizontal stacking, and overlay or alignment.
- Modifier order affects layout, drawing, and input behavior.
- Click target behavior depends on where `clickable` appears relative to sizing and padding modifiers.
- `padding` participates in measurement and layout, while `offset` moves content visually after layout.
- `remember` stores local state across recomposition; `rememberSaveable` is used for simple UI state that should survive configuration changes.
- State hoisting replaces internal state with a value parameter and callback when state must be shared or controlled by a parent.
- `LazyColumn` is the preferred primitive for growing vertical lists.
- Stable keys are useful when list items can be inserted, removed, or reordered.
- `LaunchedEffect` is for suspend work tied to composition and keys; `rememberCoroutineScope` is for launching coroutines from event handlers.
- `DisposableEffect` is for side effects that need cleanup.
- `rememberUpdatedState` lets an effect use a current callback without restarting.
- `derivedStateOf` and `snapshotFlow` support derived state and Flow conversion from Compose state reads.
- High-level click modifiers are preferred for normal clicks; lower-level `pointerInput` is used for custom gesture handling.
- `drawBehind`, `drawWithContent`, and `Canvas` cover progressively more custom drawing needs.
- `animate*AsState`, `AnimatedVisibility`, and `AnimatedContent` cover common simple animation cases.
